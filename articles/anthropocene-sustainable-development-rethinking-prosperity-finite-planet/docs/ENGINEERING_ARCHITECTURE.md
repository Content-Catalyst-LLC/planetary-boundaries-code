# Engineering Architecture

This repository treats Anthropocene sustainable development as a prosperity-within-limits diagnostics problem.

## Data layer

The SQL schema defines:

- development scenarios
- development indicators
- response capacity indicators
- sustainable prosperity scores

This supports provenance, versioning, transparency, and reproducibility.

## Analytics layer

Python and R provide social foundation gaps, ecological overshoot, boundary-adjusted pressure, response capacity, sustainable prosperity scoring, and transition-urgency diagnostics.

## Systems layer

Rust and Go can translate prosperity-within-limits scoring into typed scoring cores, APIs, and dashboard backends.

## Edge layer

C, C++, TinyML, and PYNQ scaffolds can support threshold alerts, anomaly detection, scenario simulation, and accelerated preprocessing.

## Interpretation

The architecture preserves the distinction between wellbeing, social foundations, ecological pressure, planetary-boundary pressure, governance capacity, justice capacity, resilience, efficiency, mitigation, restoration, and transition urgency.
