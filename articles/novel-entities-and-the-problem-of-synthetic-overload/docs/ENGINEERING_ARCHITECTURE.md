# Engineering Architecture

This repository treats novel entities as a synthetic-overload and governance-capacity problem.

## Data layer

The SQL schema defines:

- entity classes
- substances
- production and release profiles
- risk properties
- governance capacity
- scoring runs
- synthetic overload scores

This supports provenance, versioning, transparency, and reproducibility.

## Analytics layer

Python and R provide synthetic-overload scoring, assessment-gap analysis, scenario testing, and dashboard-ready exports.

## Systems layer

Rust and Go demonstrate how synthetic-overload scoring could become:

- a typed scoring core
- a command-line tool
- a JSON API
- a dashboard backend

## Edge layer

C, C++, TinyML, and PYNQ scaffolds show how chemical or environmental signals could be linked to threshold alerts, anomaly detection, scenario simulation, and accelerated preprocessing.

## Interpretation

The architecture preserves the distinction between production, release, persistence, mobility, hazard, exposure, monitoring, assessment status, substitution feasibility, and governance interpretation.
