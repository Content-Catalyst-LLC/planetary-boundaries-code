# Implementation Notes

This module is designed around five linked questions:

1. Where are planetary-boundary applications most vulnerable to technocratic misuse?
2. Which cases combine high ecological pressure with weak justice or legitimacy?
3. Where does downscaling create methodological fragility?
4. Which applications hide political-economic drivers behind neutral technical language?
5. How can critique-aware analytics make assumptions, uncertainty, and values inspectable?

## Metric architecture

The article body includes Python and R workflows for analytical clarity. The repository extends the same logic into SQL, Rust, Go, C, C++, TinyML, and PYNQ scaffolds.

## Caution

All scores, weights, and thresholds are illustrative. Replace them with documented, transparent, and context-specific assumptions before applied use.
