# Critiques of the Planetary Boundaries Framework

This article folder supports the Planetary Boundaries knowledge-series article on critiques of the planetary boundaries framework.

The article body includes Python and R workflows for readability and reproducible analytics. This repository extends the implementation into a broader engineering scaffold for auditability, APIs, systems programming, embedded monitoring, TinyML, and PYNQ-oriented edge workflows.

## Structure

- `python/` — critique-risk scoring and sensitivity analysis
- `r/` — dashboard-ready governance, justice, and operationalization summaries
- `sql/` — schema for critique domains, assumptions, provenance, and audit trails
- `rust/` — reliable critique-risk scoring core scaffold
- `go/` — lightweight diagnostic API scaffold
- `c/` — embedded-style threshold monitor with legitimacy/uncertainty flags
- `cpp/` — scenario simulation scaffold
- `tinyml/` — edge anomaly detection scaffold
- `pynq/` — accelerator-oriented monitoring scaffold
- `docs/` — methodological and engineering notes
- `data/raw/` — raw input data
- `data/processed/` — cleaned and transformed data
- `outputs/` — exported diagnostic scores and dashboard-ready files

## Methodological Note

The sample workflows use illustrative data. Replace placeholder values with documented indicators, transparent institutional assumptions, expert elicitation, or participatory scoring before applied use.
