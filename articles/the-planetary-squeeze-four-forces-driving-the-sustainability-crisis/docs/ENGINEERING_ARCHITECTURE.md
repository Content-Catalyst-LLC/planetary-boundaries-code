# Engineering Architecture

This repository treats the planetary squeeze as a coupled socio-ecological scenario diagnostics problem.

## Data layer

The SQL schema defines:

- squeeze scenarios
- four-force pressure indicators
- response capacity indicators
- planetary squeeze scores

This supports provenance, versioning, transparency, and reproducibility.

## Analytics layer

Python and R provide core squeeze pressure, interaction amplification, boundary-adjusted risk, response capacity, transformation urgency, and dashboard-ready exports.

## Systems layer

Rust and Go can translate squeeze scoring into typed scoring cores, APIs, and dashboard backends.

## Edge layer

C, C++, TinyML, and PYNQ scaffolds can support threshold alerts, anomaly detection, scenario simulation, and accelerated preprocessing.

## Interpretation

The architecture preserves the distinction between population pressure, affluence pressure, climate stress, ecosystem degradation, boundary pressure, governance capacity, justice capacity, adaptation, mitigation, restoration, and material efficiency.
