# Engineering Architecture

This repository treats the Great Acceleration as a coupled socio-economic and Earth-system diagnostics problem.

## Data layer

The SQL schema defines:

- socio-economic indicators
- Earth-system indicators
- annual observations
- coupled indicator pairs
- acceleration risk scores

This supports provenance, versioning, transparency, and reproducibility.

## Analytics layer

Python and R provide human activity pressure, Earth-system stress, coupling strength, boundary pressure, response capacity, transformation urgency, and dashboard-ready exports.

## Systems layer

Rust and Go can translate acceleration scoring into typed scoring cores, APIs, and dashboard backends.

## Edge layer

C, C++, TinyML, and PYNQ scaffolds can support threshold alerts, anomaly detection, scenario simulation, and accelerated preprocessing.

## Interpretation

The architecture preserves the distinction between growth, acceleration, coupling, boundary pressure, lock-in, governance, justice, mitigation, and restoration.
