# Engineering Architecture

This repository treats the origins of the planetary boundaries framework as a knowledge-architecture, framework-evolution, and governance-diffusion problem.

## Data layer

The SQL schema defines:

- framework milestones
- source documents
- conceptual lineages
- framework scores
- governance uptake
- critique and revision notes

This supports provenance, versioning, transparency, and reproducibility.

## Analytics layer

Python and R provide framework-evolution scoring, milestone comparison, influence classification, justice-gap analysis, and dashboard-ready exports.

## Systems layer

Rust and Go demonstrate how framework-evolution scoring could become:

- a typed scoring core
- a command-line tool
- a JSON API
- a dashboard backend

## Optional edge and acceleration layer

C, C++, TinyML, and PYNQ scaffolds show how classification, source triage, and document-processing workflows could be modeled for larger knowledge systems.

## Interpretation

The architecture preserves the distinction between scientific maturity, governance influence, systems depth, operational readiness, justice integration, uncertainty treatment, and public legibility.
